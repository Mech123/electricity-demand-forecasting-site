Alex's Feature Blueprint

After EDA, Alex converts observations into machine-readable signals. The objective is simple: preserve the grid's repeating rhythms (hour/day/week), capture autocorrelation (lags + rolling stats), and inject weather influence without noise.

Alex's acceptance criteria
  • Features must represent real operational patterns
  • No leakage: time-aware splits only
  • Robust scaling for learning models; target remains original scale

Feature Groups (How Alex builds “signals”)

Calendar features (Step 8)

  • Basic: hour, day_of_week, month, quarter, year
  • Cyclical encodings: sin/cos for hour, weekday, month
  • Operational flags: is_weekend, time_of_day, season
  • Degree-days: heating_degree_days, cooling_degree_days

Purpose: represent predictable operational rhythm so the model does not “re-learn” calendars.

Lagged & rolling features (Step 9)

  • Target lags: 1, 2, 3, 6, 12, 24, 48, 168
  • Rolling statistics: mean/std (multiple windows)
  • EMA features: smooth short-term noise
  • Weekly seasonality: same hour last week (lag=168)

Purpose: encode autocorrelation structure (the core of short-horizon forecasting).

Weather proxies (Step 10)

  • solar_proxy (from solar generation)
  • wind_proxy (from wind generation)
  • temp_proxy (derived from solar + seasonal context)

Purpose: keep weather influence, reduce raw noise, and make learning more robust.

Encoding + normalization (Steps 11–13)

  • One-hot encoding: time_of_day, season
  • Infinity handling: no infinite values detected
  • RobustScaler applied to 82 features

Purpose: stable training for LSTM / hybrid models while preserving interpretability of target.
